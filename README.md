# 信息系统项目管理师

信息系统项目管理师（Certified Information Systems Project Manager，CISPM）是由中国信息产业协会（China Information Industry Association，CIIA）主管，中国电子信息行业发展研究院（China Academy of Information and Communications Technology，CAICT）承办的国家级IT专业认证项目。该认证旨在对信息系统项目管理人员的职业素质、技能和能力进行评估和认证，以提升信息系统项目管理人员的职业水平和整体素质。

CISPM认证考试主要涵盖信息系统项目管理的基本理论和方法、项目计划与控制、质量管理、风险管理、团队管理、沟通管理、合同管理等方面的知识。考试分为两个阶段，分别是理论考试和综合实践考试。通过考试并取得认证资格的人员，可以获得国家认可的信息系统项目管理师证书。

获得CISPM认证的人员可以在信息系统项目管理方面展示自己的专业能力和技能，为个人职业发展和企业业务发展提供有力支持。同时，CISPM认证也有助于推动信息系统项目管理领域的规范化和专业化发展。

认证考试可能会因地区、时间等因素而略有不同，但一般来说，根据官方出版物《信息系统项目管理师》第二版的介绍，该认证考试的题型和数量如下：

1. 单选题：共70道题，每道题目的分值为1分。
2. 多选题：共20道题，每道题目的分值为1分。
3. 填空题：共10个空，每个空的分值为1分，总分值为10分。
4. 论述题：共2道题，每道题目的分值为20分，要求在规定的时间内完成，并且答案要简明扼要，逻辑清晰、语言流畅，文字不少于500字。

具体考试题目的数量和分值可能会有所变化，建议考生参考官方通知和考试指南，以便更好地备考。

“信息系统项目管理师”认证考试的论文部分主要考察考生在实践中解决信息系统项目管理问题的能力，要求考生根据题目要求，结合实际情况，撰写一篇不少于2000字的论文。具体要求如下：

1. 题目要求
   考生需要根据考试要求的题目要求，从实践中选择一个信息系统项目管理问题，进行分析和解决。
2. 论文长度
   论文的长度不少于2000字，建议在3000字左右。不得少于2000字，否则将被扣分。
3. 论文结构
   论文应包括题目、摘要、引言、主体、结论和参考文献等部分。其中，引言部分应包括选题背景、研究目的和意义等；主体部分应包括问题分析、解决方法、实施步骤等；结论部分应总结论文研究成果，提出展望和建议等。参考文献要按照规范格式列出，确保引用资料的可查性和准确性。
4. 论文质量
   论文应遵循论文写作的基本规范，包括语言简练、表述清晰、逻辑严密等。同时，论文也应突出考生在解决实际问题中的创新性和实践性。评分标准包括问题解决的深度和广度、解决方法的创新性、实施步骤的合理性和可操作性、结论的正确性和严谨性等。

论文部分是“信息系统项目管理师”认证考试中非常重要的一部分，考生需要认真准备、认真思考，并结合实际情况撰写高质量的论文。

# 高频知识点

## 项目管理概念和理论

1. 项目管理概念：
   项目是一项独特的、有限的且具有风险的活动，旨在产生预定的结果或效果，需在特定的时间、预算和资源约束下进行计划、实施和控制。
   项目管理是一种应用知识、技能、工具和技术来满足项目要求的综合性管理过程，包括项目规划、执行和控制等环节。

2. 项目管理理论：
   项目生命周期理论：项目生命周期是指从项目开始到结束的整个过程，通常分为五个阶段：启动、规划、执行、监控和收尾。不同阶段具有不同的特点和重点，需要采取不同的管理策略和方法。
   WBS（工作分解结构）理论：WBS是将项目分解为不同的工作包和任务，形成一个层级结构，方便对项目进度、资源、成本等方面进行管理和控制。
   PERT/CPM理论：PERT/CPM是常用的项目进度管理方法，包括网络图和关键路径分析等，可以对项目进度进行计算和分析，找出关键路径和节点，进行进度控制。
   项目管理知识体系（PMBOK）理论：PMBOK是项目管理领域的全球通用标准，包括项目整体管理、范围管理、时间管理、成本管理、质量管理、资源管理、风险管理、采购管理、沟通管理、干系人管理等方面的知识体系。

3. 示例：
   在项目启动阶段，需要确定项目目标和可行性，编制项目章程和初步的项目计划。在这一阶段，需要采用 SWOT 分析等方法对项目进行评估和分析，找出项目的优势、劣势、机会和威胁，并根据分析结果制定相应的管理策略和计划。
   在项目执行阶段，需要对项目计划进行具体实施和监控，分配任务和资源，跟踪进度和质量，协调项目成员之间的合作，及时解决问题和风险。
   在项目收尾阶段，需要进行项目总结和评估，提交项目成果和报告，评估项目绩效和成果，并对项目经验进行总结和归档。

## 项目管理流程和方法

1. 项目管理流程
   项目管理流程是指项目在启动、规划、执行、监控和收尾阶段中，需要按照一定的流程顺序进行管理。常见的项目管理流程包括：
   * 项目启动阶段：确定项目的背景、目标和可行性分析，制定项目章程和项目管理计划。
   * 项目规划阶段：明确项目的范围、时间、成本和质量要求，制定详细的项目计划，制定风险管理计划、质量管理计划和人力资源管理计划。
   * 项目执行阶段：按照项目计划进行任务分配、资源调配、进度控制和质量保障，以达到项目目标。
   * 项目监控阶段：对项目执行过程进行监督、控制和改进，及时发现和解决问题。
   * 项目收尾阶段：完成项目目标，进行验收、交付、总结和归档。

2. 项目管理方法
   项目管理方法是指用于完成项目管理流程的具体方法和技术。常见的项目管理方法包括：
   * WBS（工作分解结构）：将项目分解为若干个可管理的工作包，有助于任务分配和进度控制。
   * PERT/CPM（网络计划）：通过绘制项目活动之间的逻辑关系网络，计算出项目的关键路径和最短工期，有助于进度控制。
   * 敏捷方法：一种以迭代、逐步和自适应方式进行项目管理的方法，能够快速响应变化和提高客户满意度。
   * SCRUM方法：敏捷方法的一种，强调团队合作、产品迭代和持续改进。
   * 风险管理：识别、评估、规避和应对项目风险，有助于保障项目进度和质量。
   * 质量管理：按照质量标准和要求进行质量控制和评估，保障项目的产品和服务质量。
   * 沟通管理：及时、准确、有效地进行项目信息沟通和协调，有助于提高项目团队合作和满意度。

3. 示例: 
   * 项目生命周期管理：指的是从项目启动到项目结束的整个过程。它包括项目规划、项目执行、项目监控和项目收尾四个阶段。在每个阶段中，需要制定相应的计划和措施来确保项目的成功。
   * 里程碑法：是一种基于项目关键节点的管理方法。它将项目分解为多个子任务，并为每个子任务设定一个具体的完成时间点，称之为里程碑。通过监控里程碑的完成情况，可以及时发现并解决问题，确保项目进度和质量。
   * 精益项目管理：是一种以价值为导向的管理方法。它强调在整个项目生命周期中不断优化过程，降低成本，提高质量。通过精益项目管理，可以更好地满足客户需求，提升企业竞争力。
   * 敏捷项目管理：是一种注重灵活性和响应能力的管理方法。它强调快速响应变化，不断交付有价值的成果。通过敏捷项目管理，可以更好地适应市场变化，提高项目成功率。

例如：在一个软件开发项目中，使用敏捷项目管理方法，将项目分解为多个迭代周期，每个周期持续两周。在每个迭代周期中，团队将开发一些有价值的功能，并及时反馈客户意见。通过迭代周期的不断迭代，最终交付符合客户需求的软件产品。

## 项目计划和调度

项目计划和调度是信息系统项目管理中至关重要的一环，其目的是确定项目执行过程中需要遵循的计划和时间表，以确保项目能够按时交付、在预算范围内完成，并且达到质量和客户的要求。下面是项目计划和调度的主要内容和步骤：

1. 确定项目目标和范围：项目计划和调度的第一步是明确项目的目标和范围。这包括了确定项目的愿景、目标、可交付成果、项目限制条件、所需资源等等。
2. 制定项目计划：在明确了项目目标和范围之后，就需要制定一个详细的项目计划。项目计划包括了时间、质量、资源、成本等方面的考虑，以及项目执行的方法和流程。
3. 识别项目依赖关系：在项目计划制定过程中，需要识别出项目各个活动之间的依赖关系，以便能够准确地规划项目时间表和资源分配。
4. 确定项目资源：在项目计划中还需要确定所需的人力、物资和资金等资源，并制定相应的资源计划和管理措施。
5. 制定项目时间表：制定项目时间表是项目计划和调度的重要一环。在制定时间表的过程中，需要考虑到项目中的每个活动的前置任务和后置任务，并基于项目的目标、资源、约束条件等因素来制定时间表。
6. 制定项目预算：制定项目预算是确保项目在可控的成本范围内完成的关键。项目预算需要考虑到项目的规模、工期、人力、材料和设备等方面的因素，并进行适当的风险分析和成本控制。
7. 项目调度和跟踪：制定好项目计划和时间表后，就需要对项目进行调度和跟踪，以确保项目在预定时间内完成。项目调度和跟踪涉及到各个方面的工作，包括了任务分配、资源调配、时间控制、质量监督等等。

让我们通过一个实际的例子来说明项目计划和调度的具体步骤和技巧。假设我们要实施一个新的在线购物网站，以下是项目计划和调度的示例：

1. 确定项目目标和范围：我们的目标是创建一个全新的在线购物网站，提供高品质的产品和优质的服务，以吸引更多的用户。
   网站将包括以下功能：
   * 注册和登录功能
   * 商品浏览和搜索功能
   * 购物车和结算功能
   * 用户账户和个人信息管理功能
2. 制定项目计划：确定项目阶段和关键里程碑，并制定详细的项目计划。我们将采用敏捷开发方法，并将项目分为以下阶段：
   * 需求分析和设计阶段（2周）
   * 开发和测试阶段（12周）
   * 上线和运营阶段（持续进行）
3. 确定项目资源和预算：
   * 确定所需的人力资源、技术资源和预算，以确保项目成功实施。我们需要开发人员、测试人员、UI设计师、数据库管理员等。
4. 制定项目调度：制定项目调度计划，确定每个任务的开始和结束时间，以确保项目按时交付。我们将使用甘特图来展示项目调度。
5. 监控项目进展：监控项目进展并及时采取措施，以确保项目按时完成。我们将每日更新项目进度，并进行风险管理和问题解决。

通过以上步骤和技巧，我们可以成功制定和实施项目计划和调度，确保项目按时完成并达到预期目标。
